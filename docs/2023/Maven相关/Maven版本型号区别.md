# Maven版本型号区别

> 2025-12-18 14:13:59

Maven 中三种版本号类型（SNAPSHOT、RELEASE、普通数字版）的核心区别

## 三种版本号类型的核心区别

从定义、用途、Maven 行为三个维度来拆解：

### 1. SNAPSHOT（快照版本）

- **定义**：版本号以 `-SNAPSHOT` 结尾（如 `1.0.0-SNAPSHOT`），代表**开发中的不稳定版本**。
- **核心用途**：用于**开发阶段**，比如团队内部迭代、功能未最终定稿时使用。
- **Maven 核心行为**：
    - 每次构建时，Maven 会优先从仓库（本地/远程）检查是否有**更新的快照版本**（哪怕本地已有该版本，也会尝试拉取最新的）；
    - 发布到仓库时，Maven 会自动生成带时间戳的唯一版本（如 `1.0.0-20251218.080000-1`），而非直接用 `1.0.0-SNAPSHOT` 存储；
    - 不建议在生产环境使用，因为依赖可能随时变化，导致构建结果不一致。

### 2. RELEASE（正式发布版本）

- **定义**：版本号以 `-RELEASE` 结尾（如 `1.0.0-RELEASE`），是 Maven 约定俗成的**正式发布版本标识**（并非 Maven
  强制语法，更多是行业惯例）。
- **核心用途**：用于**测试完成、功能稳定**的正式发布版本，对外提供的稳定依赖。
- **Maven 核心行为**：
    - Maven 对 `-RELEASE` 没有特殊语法处理，本质上和“普通数字版”一样，只是语义上明确这是正式版；
    - 一旦发布到仓库，**不会被覆盖**，本地拉取一次后就不会再主动更新（除非手动删除本地仓库的该版本）；
    - 生产环境优先使用这类版本，保证依赖的稳定性。

### 3. 普通数字版本（纯数字/数字+字符，无后缀）

- **定义**：版本号仅由数字、点、横杠（非 SNAPSHOT/RELEASE）组成（如 `1.0.0`、`2.1.5-beta`、`3.0.1-rc1`），是最基础的版本格式。
- **核心用途**：可灵活用于正式版、测试版、候选版等，完全由开发者定义语义（如 `beta` 代表测试版，`rc` 代表发布候选版）。
- **Maven 核心行为**：
    - 与 `-RELEASE` 一致，Maven 视为“稳定版本”，拉取一次后不会主动更新；
    - 无特殊语法规则，仅作为版本标识，语义完全由团队约定。

## 关键对比表

| 特性        | SNAPSHOT 版本 | RELEASE 版本  | 普通数字版本         |
|:----------|:------------|:------------|:---------------|
| 稳定性       | 不稳定（开发中）    | 稳定（正式发布）    | 灵活（依团队定义）      |
| Maven更新策略 | 每次构建检查最新版本  | 仅首次拉取，后续不更新 | 仅首次拉取，后续不更新    |
| 仓库存储形式    | 带时间戳的唯一版本   | 直接存储版本号     | 直接存储版本号        |
| 适用场景      | 开发/测试阶段     | 生产/正式发布     | 测试版/候选版/自定义正式版 |

## 总结

1. **SNAPSHOT** 是开发中的“动态版本”，用于团队内部迭代，Maven 会自动拉取最新版；
2. **RELEASE** 是约定俗成的“正式版本”，Maven 视为稳定版，不会主动更新；
3. **普通数字版本** 是基础格式，无 Maven 特殊处理，语义由团队自定义（可替代 RELEASE 作为正式版）。

核心原则：开发阶段用 SNAPSHOT，正式发布用 RELEASE 或纯数字版，生产环境坚决避免依赖 SNAPSHOT 版本。
