# Linux检测端口占用

> 2026-02-12 14:01:03

* 命令(以检测9150端口为例)

```bash
netstat -tulpn | grep 9150
```

## 命令拆解与详细解释
这条命令由两部分组成，通过管道符 `|` 连接，我们一步步拆解：

### 1. 核心命令：`netstat -tulpn`
`netstat` 是 Linux 系统中查看网络状态的基础工具，后面的参数是功能筛选：
| 参数 | 含义 |
|------|------|
| `-t` | 只显示 **TCP** 协议的连接/监听端口 |
| `-u` | 只显示 **UDP** 协议的连接/监听端口 |
| `-l` | 只显示 **处于监听状态** 的端口（不是已建立的连接） |
| `-p` | 显示占用该端口的 **进程ID（PID）** 和进程名称 |
| `-n` | 以 **数字形式** 显示IP和端口（不解析域名/服务名，速度更快） |

简单说，`netstat -tulpn` 就是「列出所有正在监听的TCP/UDP端口，并显示每个端口对应的进程信息」。

### 2. 过滤筛选：`grep 9150`
`grep` 是 Linux 中的文本过滤工具，`grep 9150` 会从上一步 `netstat` 的输出结果中，只保留包含「9150」的行——也就是只显示和 9150 端口相关的信息。

## 实际使用场景与输出示例
### 典型输出（以Tor服务默认占用9150端口为例）：
```
tcp        0      0 127.0.0.1:9150          0.0.0.0:*               LISTEN      1234/tor
udp        0      0 127.0.0.1:9150          0.0.0.0:*                           1234/tor
```
输出字段解读：
- `tcp/udp`：协议类型
- `127.0.0.1:9150`：监听的IP和端口（127.0.0.1表示仅本机可访问）
- `LISTEN`：端口处于监听状态
- `1234/tor`：占用端口的进程PID是1234，进程名是tor

### 无输出的情况：
如果执行后没有任何结果，说明 **9150端口当前没有被任何进程监听**（可能端口未启用、进程未启动，或被防火墙屏蔽）。

## 注意事项
- 执行该命令通常需要 **root权限**（加 `sudo`），否则 `-p` 参数可能无法显示进程信息（显示 `--`）；
- 部分新系统中 `netstat` 已被 `ss` 替代，等效命令：`ss -tulpn | grep 9150`。

## 总结
1. 这条命令的核心作用是：**查询Linux系统中9150端口是否被监听，以及占用该端口的进程ID和名称**；
2. `-tulpn` 是关键参数组合，实现「筛选TCP/UDP监听端口+显示进程+数字格式」的效果；
3. `grep 9150` 是精准过滤，只保留9150端口的相关信息。
